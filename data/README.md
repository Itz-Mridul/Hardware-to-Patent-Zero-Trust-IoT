# Data Directory

This directory stores runtime data generated by the Zero-Trust IoT Security Gateway:

## Contents

| File | Generated By | Purpose |
|------|-------------|---------|
| `heartbeats_real.csv` | `collect_training_data.py` | Real ESP32 heartbeat samples for ML training |
| `heartbeats_attack.csv` | `software_attacker.py` | Simulated attack samples for ML training |
| `merged_training.csv` | `merge_datasets.py` | Combined dataset for CNN-LSTM model training |

## Important Notes

- **Do NOT commit** `.db`, `.h5`, or `.pkl` files to git
- Training data CSVs may be committed for reproducibility
- The `security.db` database is created at runtime in `pi_backend/`
- ML model files (`.h5`, `.pkl`) should be uploaded via GitHub Releases

## Generating Training Data

```bash
# 1. Let ESP32-CAM run for 30+ minutes to collect real heartbeats
python3 pi_backend/collect_training_data.py

# 2. Generate attack samples
python3 tests/software_attacker.py

# 3. Merge datasets
python3 pi_backend/merge_datasets.py

# 4. Train the CNN-LSTM model
python3 pi_backend/train_authentication_model.py
```
